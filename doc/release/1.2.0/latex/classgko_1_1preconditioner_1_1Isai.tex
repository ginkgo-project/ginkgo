\doxysubsection{Detailed Description}
\subsubsection*{template$<$isai\+\_\+type Isai\+Type, typename Value\+Type, typename Index\+Type$>$\newline
class gko\+::preconditioner\+::\+Isai$<$ Isai\+Type, Value\+Type, Index\+Type $>$}

The Incomplete Sparse Approximate Inverse (I\+S\+AI) Preconditioner generates an approximate inverse matrix for a given lower triangular matrix L or upper triangular matrix U. 

Using the preconditioner computes $aiU * x$ or $aiL * x$ (depending on the type of the \mbox{\hyperlink{classgko_1_1preconditioner_1_1Isai}{Isai}}) for a given vector x (may have multiple right hand sides). aiU and aiL are the approximate inverses for U and L respectively.

The sparsity pattern used for the approximate inverse is the same as the sparsity pattern of the respective triangular matrix.

For more details on the algorithm, see the paper \href{https://doi.org/10.1016/j.parco.2017.10.003}{\texttt{ Incomplete Sparse Approximate Inverses for Parallel Preconditioning}}, which is the basis for this work.

\begin{DoxyNote}{Note}
G\+PU implementations can only handle the vector unit width {\ttfamily width} (warp size for C\+U\+DA) as number of elements per row in the sparse matrix. If there are more than {\ttfamily width} elements per row, the remaining elements will be ignored.
\end{DoxyNote}
